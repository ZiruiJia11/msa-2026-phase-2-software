## Current Figma Structure

The Figma file has been manually organised into two main design areas:

### FitQuest 01 - Pixel Assets

This page is used for FitQuest visual assets and gamification ideas.

Planned and current asset direction:

- Original pixel-style FitQuest character concepts.
- Small exercise action frame ideas for workout categories.
- Sprite-style inspiration for future frontend animation.
- Visual references for achievements, quest feedback, and progress moments.

The pixel assets should stay original to FitQuest and should not copy specific copyrighted character designs.

### FitQuest 02 - UX Screens

This page is used for the main application screen direction.

Planned and current screens:

- Dashboard
- Quests
- Progress
- Achievements
- Settings

These screens should guide the real React TypeScript frontend implementation.

## Design Direction

FitQuest should feel energetic, game-like, and simple to understand.

The UI should prioritise:

- Clear left-side navigation on desktop.
- Fast scanning of XP, level, streak, and quest progress.
- Vibrant colours that feel active and motivational.
- Mobile-friendly layouts.
- Clear quest cards and actions.
- Achievement and progress visuals that support the gamification theme.

The app should avoid:

- Medical advice.
- Personalised workout prescriptions.
- Nutrition, calorie, injury, or healthcare recommendations.
- Overly complex fitness analytics that distract from the MVP.

## Implementation Handoff

The Figma design is the baseline for future frontend UI work.

The next frontend implementation steps should be:

1. Align the app shell and navigation with the Figma UX direction.
2. Update the Quest Board visual style to match the new FitQuest design language.
3. Add placeholder routes or page structures for Dashboard, Progress, Achievements, and Settings.
4. Use the pixel asset direction as visual inspiration before implementing full animation.
5. Keep each UI improvement in a small, focused commit.
